# 60 — ドキュメント同期

実装が完了状態に近づいたら、以下の同期を必須にする。同期漏れは Evaluator が差し戻す。

## 同期対象

- **feature-contract**: 実装で確定した API 契約・受け入れ条件を契約側へ反映する。
  契約と実装が食い違ったまま completed にしない。
- **README / docs**: 利用者向けの手順・仕様変更を反映する。
- **`.claude/project-profile.md`**: 追加・変更された環境変数名、ドメイン用語、
  リポジトリ規約を反映する。固有情報はここにだけ書く。

## 原則

- ドキュメント更新は「後でやる」を許さない。DoD（done-definition）の 1 項目として扱う。
- 契約変更を伴う場合は Planner 経由で行う（実装者が勝手に契約を書き換えない）。
- 変更概要（何を・なぜ）を feature-contract または review-report に残す。
